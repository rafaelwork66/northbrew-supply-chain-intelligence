#Northbrew Supply Chain Intelligence Dashboard
Overview

A Power BI supply chain analytics project analysing inventory, supplier reliability, stockout risk, and procurement cost exposure using a simulated dataset.

Business Problem

Northbrew needs better visibility into inventory concentration, supplier performance, procurement cost, and operational risk.

Objectives
Build a supply chain KPI framework.
Create a star-schema data model.
Develop Power BI dashboards for different stakeholders.
Translate findings into management actions.

Tools Used :
1. Power BI
2. DAX
3. Power Query
4. Data modelling
5. imulated supply chain dataset
6. AI-assisted analysis and dashboard planning

Key Insights :
 
Insight 1

Inventory value is high, but stockout risk still exists in Mobile and Accessories.

Meaning:

The issue is not simply “not enough stock.” It is likely an allocation or replenishment problem.

Insight 2

Networking has the highest inventory value, but not the highest stockout pressure.

Meaning:

Networking may need overstock monitoring rather than urgent replenishment.

Insight 3

Mobile and Accessories drive most stockout risk.

Meaning:

These categories should be prioritised for safety stock and replenishment review.

Insight 4

Supplier fill rate does not vary much.

Meaning:

Supplier performance should be reviewed using delivery delay, defect rate, and landed cost, not fill rate alone.

Insight 5

Freight cost creates procurement exposure.

Meaning:

Some suppliers or countries may look acceptable on purchase cost but become expensive after freight is included.

$Recommendations

Management should:

Review safety stock for Mobile and Accessories.
Monitor Networking inventory for possible overstock.
Review high-spend suppliers with delivery delays or quality issues.
Investigate freight-heavy supplier routes.

Limitations

This project uses simulated data, so the findings are designed for portfolio demonstration rather than real operational decision-making.
